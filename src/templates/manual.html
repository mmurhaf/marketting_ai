📖 User Manual

Everything you need to know to run your social media marketing pipeline — from scraping product images to publishing reels on Instagram, Facebook, and TikTok.

🌐 What This System Does

Marketing AI is a self-hosted automation tool with a browser-based dashboard. It handles the full content pipeline for social media:

🕷️
Scrape Images Automatically download product images from your website. Duplicate images are detected and skipped using MD5 hashing.
🎬
Create Reels Turn scraped images into 9:16 portrait MP4 videos (1080×1920) with fade transitions and optional background music.
🖼️
Create Post Images Generate 1:1 square images (1080×1080) with text overlays (title + description) for carousel posts.
📤
Publish to Social Media Send your content to Instagram, Facebook, and TikTok directly from the browser. Choose platforms and write a caption per post.
🎵
Background Music Upload MP3/WAV/OGG files and attach them to reels. Batch-add music to all silent videos with one button.
⚙️
Configurable & Schedulable All credentials and settings live in one place. Run entirely headless on a schedule with no browser needed.

🔄 Typical Daily Workflow ↑ top

Follow these steps in order each time you want to post new content:

1
Scrape new images Dashboard → Run Scraper Now, or Editor → upload your own images.
2
Select images Open the Editor, tick the images you want to use.
3
Choose output type Pick Reel (video) or Post (image). Optionally select a music track.
4
Create & preview Click Create. Preview the result in the right panel.
5
Write caption & publish Check your platforms, type a caption, and click Publish.
Tip — Let the automation do it all: If you just want everything to run automatically, click Run Scraper Now → Create Reel Now → Publish Latest Reel on the Dashboard one after the other. Or use headless mode to run the entire pipeline on a schedule.

🏠 Dashboard ↑ top

The Dashboard (/) is the quick-action control panel. All buttons post to the server and show the result in the Live Status console at the bottom.

Quick Actions

ButtonWhat it does
Run Scraper Now Visits your configured Image Source URL and downloads up to N new product images (configurable in Settings). Already-downloaded images are skipped automatically.
Create Reel Now Randomly selects up to 5 images from the image library and generates a 9:16 MP4 reel with a fade transition. Background music is added automatically if any music files are uploaded.
Publish Latest Reel Finds the most recently modified video in assets/output/ and publishes it to Instagram and Facebook using your configured credentials and default caption.

Tools

ButtonWhat it does
Auto-Add Music to Silent Videos Scans every video in assets/output/, detects ones without an audio track, and merges a randomly chosen music file from assets/music/. The new file is saved with a _music suffix so the original is preserved.
Restart Server Gracefully relaunches the Flask server process. Use this after editing config files manually. Wait ~5–10 seconds then refresh the page.
The Live Status box at the bottom logs every action's output in real time. If an action fails, the error message is shown there.

✏️ Editor ↑ top

The Editor (/editor) gives you full control over content creation. It is split into two columns: image selection on the left, creation & publishing on the right.

Step 1 — Select Images

  • All images in assets/images/ are shown in the grid.
  • Click a thumbnail (or its checkbox) to select it. Selected images get a blue border. Click again to deselect.
  • Use Load More to page through large libraries.
  • Click + Upload Images to upload your own images from your computer (JPG, PNG, GIF accepted).
  • Click the red ✕ button on any image to permanently delete it from the server.

Step 2 — Choose Output Type

TypeOutput formatBest for
Reel (Video)MP4, 1080×1920, 24 fpsInstagram Reels, TikTok, Facebook Reels
Post (Image)JPG, 1080×1080Instagram Feed, Facebook Posts

Step 2b — Background Music (Reels only)

  • The Music dropdown lists all files in assets/music/.
  • Select a track or leave it on Auto (Random) to let the system pick one.
  • To add music files, click + Upload Music (MP3, WAV, OGG accepted).

Step 3 — Create

Click Create Reel or Create Post. The server generates the file and the preview panel on the right shows the result. Reel generation may take 30–60 seconds depending on image count and hardware.

Step 4 — Publish

  • Check one or more platforms (Instagram, Facebook, TikTok).
  • Edit the Caption text area — it is pre-filled with your default caption from Settings.
  • Click Publish. Results appear immediately below the button.
Publishing simulation: Until you implement real API credentials in publisher.py, publishing will simulate the upload. See the Setup Guide to connect real accounts.

📂 Uploading Your Own Files ↑ top

Upload images

  1. Go to the Editor.
  2. Click + Upload Images (top-right of the image grid).
  3. Select one or more image files (JPG, PNG, GIF).
  4. The images are saved to assets/images/ and appear in the grid immediately.

Upload music

  1. Go to the Editor.
  2. Click + Upload Music.
  3. Select an audio file (MP3, WAV, or OGG).
  4. The file is saved to assets/music/ and appears in the music dropdown.
You can also place files directly into assets/images/ and assets/music/ via your file manager — the server picks them up instantly without a restart.

🎵 Background Music Guide ↑ top

The system uses background music in two ways:

MethodWhen it's used
Auto (random) When you create a reel and leave the music dropdown on Auto, or use Dashboard → Create Reel Now. A random file from assets/music/ is selected.
Manual selection In the Editor, pick a specific track from the dropdown before clicking Create Reel.
Batch add Dashboard → Auto-Add Music to Silent Videos — adds a random track to every video that has no audio, saving new files with a _music suffix.
Music is looped if it is shorter than the video, and fades out over the last 2 seconds. Only files in assets/music/ are used — system audio files are not touched.

📋 Server Logs ↑ top

The Logs page (/logs) shows the rotating Flask server log stored at logs/server.log.

  • The log captures all HTTP requests, API calls, scraper output, and errors.
  • Log file rotates at 100 KB (1 backup kept). Older entries are discarded automatically.
  • Refresh the Logs page to see the latest entries — it is not a live stream.
  • If you need to diagnose a failed task, check the log immediately after the failure — the exact Python traceback is recorded.

🤖 Headless / Scheduled Mode ↑ top

Instead of the web dashboard, you can run the entire pipeline automatically on a schedule using main.py.

How to start it

  1. Open a terminal in the project root.
  2. Activate your virtual environment: .venv\Scripts\activate (Windows) or source .venv/bin/activate (Linux/macOS).
  3. Run: python src/main.py

What it does

  1. Runs once immediately on startup.
  2. Then repeats every day at 10:00 AM automatically.
  3. Each run: scrapes new images → creates a reel → publishes to Instagram & Facebook.
Running as a background service: On Windows you can use Task Scheduler to launch python src/main.py at startup. On Linux, use systemd or cron.
Headless mode uses your saved credentials from data/config.json. Make sure you have configured them via the Settings page before running headless.

🔧 Troubleshooting ↑ top

Common issues

ProblemLikely cause & fix
Reel creation fails immediately ffmpeg is not installed or not on your system PATH. Install it from ffmpeg.org and restart the server.
Scraper downloads 0 images The source URL may block scrapers, or all images are already downloaded (deduplication). Try a different URL, or clear data/history.json to re-download existing images.
Publish returns "Failed — no credentials" Credentials are missing or blank. Go to Settings, enter your tokens/passwords, and save. Use the Test buttons to verify.
Music not added to reel No files in assets/music/. Upload at least one MP3/WAV/OGG via the Editor.
Server won't start — port in use Another process is already using port 5000. Stop it, or edit the last line of src/server.py to use a different port: app.run(port=5001).
Text is cut off on post images Titles longer than ~2 lines are wrapped automatically but capped at 2 lines per field. Shorten the title in data/metadata.json or use the Post type with shorter alt text on your source images.
Page loads but actions do nothing Check the Server Logs for a Python traceback. A missing dependency is the most common cause — run pip install -r requirements.txt.
